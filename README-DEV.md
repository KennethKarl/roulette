# 세이프닥 웰컴 쿠폰팩 룰렛 — 개발 핸드오프

라이브: https://kennethkarl.github.io/roulette/ (GitHub Pages, 정적)

## 구성
- `index.html` — **라이브 페이지 정본** (단일 파일, HTML/CSS/JS 인라인, 외부 의존성은 Pretendard 폰트 CDN뿐)
- `assets/` — 섹션별 이미지 슬롯(플레이스홀더) + 규격 문서 `SLOTS.md`
- `logo.svg`, `.nojekyll` — 로고 / Pages Jekyll 비활성
- `index2.html` — 별개 변형("매일 한 번 뽑기"). 이 건과 무관, 참고용

## 동작 요약 (index.html)
- **진료과 선택**(바텀시트) → **START** 클릭 → 룰렛 회전 → **웰컴 쿠폰팩 당첨**
- **당첨 확률 100%** (`pick()`가 항상 웰컴 반환, 꽝 없음)
- **near-miss 연출**: 멈춤 없는 단일 감속으로 꽝 경계를 천천히 넘어 웰컴 '끝자락'(꽝 경계 1.5°)에 안착
- **1회 참여 제한**: `localStorage["safedoc_roulette_issued"]` (재방문·새로고침에도 유지). 유의사항의 "발급 이력 초기화"로 리셋(데모용)
- **카운트다운**: 접속 시점 +7일 자정 기준 실시간 계산
- **딥링크**: 당첨 후 `병원 보러 가기` → `safedoc.io/hospital/search/keyword?...` (선택 진료과 필터). 진료과 ID는 치과만 확보, 나머지는 `DEPTS[]`의 `id` 채우면 정확도↑

## 이미지 슬롯 (디자이너 → 개발 반영)
카운트다운·룰렛 회전·버튼·모달은 이미지 위 **라이브 오버레이**로 유지되고, 나머지 시각 섹션은
`assets/*.png` 슬롯으로 교체 가능. **규격·비워둘 영역·룰렛 세그먼트 배치 제약은 `assets/SLOTS.md` 참고.**
> ⚠ 룰렛 원판(`wheel.png`) 교체 시 세그먼트 순서/각도 유지 필수(웰컴↔꽝 인접) — 안 그러면 near-miss가 깨짐.

## 배포
정적 파일이라 그대로 웹서버/Pages에 올리면 됨. 빌드 과정 없음.
로컬 확인: `python3 -m http.server` 후 브라우저로 접속 (파일 직접 열기는 폰트/이미지 경로만 주의).
